PenTest Homelab using Metasploit and Greenbone OpenVAS

Configured a Windows 7 Virtual Machine with an initially unknown IP to test Penetration Testing techniques against. Utilized NMAP scans, Greenbone OpenVAS scans, and Metasploit to find vulnerabilities and gain access to the system.

  • Utilized a base Kali Virtual Machine to install Greenbone OpenVas and establish a Metasploit workspace on.
  • Performed an NMAP aggressive scan of entire network to find the target machines IP address.
  • Performed both UDP and TCP NMAP scans, as well as a scan of all ports.
  • Performed a full OpenVAS scan of the machine to reveal the services being ran.
  • Imported OpenVAS XML into a Metasploit workspace that then showcased a vulnerability in the VULNS table for 'Elasticsearch' which the system was running an outdated version of.
  • Targeted the ElasticSearch version with a search inside the Metasploit database, and was able to run a Dynamic Script Arbitrary Java Execution exploit that gained me basic access to a meterpeter shell.
  • Utilized the meterpeter shell to manually download a hashdump utility onto the system that gave me a hashdump of the passwords . Utilization of John The Ripper gave me system access with a privileged accounts credentials.

PenTest School-Homelab exploiting vulnerable Windows 7 machine

Configured a Windows 7 Virtual Machine with Immunity Debugger to test the effects of a custom made shellcode exploit targeting an outdated version of SLMail and attempting to cause a Reverse Shell attack.

  • Utilizing msfvenom on a base Kali VM, I generated a reverse TCP shellcode that would be able to be sent to my target Windows VM and cause the exploit.
  • A python script was provided that simplified the process of sending the shellcode, but a crucial part was initializing the shellcode with a nopslide that totals the value of 16 nops in the script
  • When ran, the shellcode would overflow the SLMail service and generate a reverse shell on the Kali machine allow me full access to the Windows VM.
  • Further testing had me utilizing the msfvenom commands to insert a nopslide directly into the code being generated and sent to the Windows 7 VM, as well as altering it so the SLMail service would continue to run after shellcode exit.
